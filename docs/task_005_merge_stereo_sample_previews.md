# Task 005: Merge Per-Wave Stereo and Sample-Based Previews

## Layout change

The old standalone **"5. Stereo Space Per Wave"** box was removed. Its controls now live in the first wave-mixing section, renamed **"1. Mix Wave Shapes and Place Them in Stereo"**.

Each waveform row now keeps its related controls together:

- wave name and icon
- sample-based mini wave preview
- start loudness
- end loudness
- change time
- per-wave ear place / pan
- per-wave ear spread / width
- per-wave ear dance
- compact left/right ear wave previews

The global stereo controls remain in the separate **"4. Stereo Space"** section because they still affect the whole mix, while the per-wave stereo shaping is now adjacent to the per-wave mix controls.

## Preview generation

Mini previews are generated by `build_wave_preview_samples()`, a lightweight pre-generation sampler. It does **not** call the full audio renderer and does **not** run Paulstretch or export processing.

The helper builds short cached arrays for each wave:

- `mono` for the main wave contour preview
- `left` and `right` for compact stereo previews
- `metadata` for active state, amplitude, pan, width, and dance

The preview sampler reuses the same oscillator shape helper used by audio generation (`waveform_from_phase`) and applies compact versions of the same envelope and panning math. `paintEvent()` only draws cached arrays and scrolls them during playback/live-loop motion.

## Reflected effects

The previews currently reflect:

- waveform shape: sine, triangle, sawtooth, and square
- per-wave start and end loudness envelope
- per-wave change time for how quickly the loudness envelope reaches its end value
- silent/off wave state through flat or near-flat samples
- global stereo start/end pan curve in stereo previews
- global stereo width in stereo previews
- global auto-pan depth and speed in stereo previews
- per-wave pan / ear place in left/right emphasis
- per-wave width / ear spread in left/right emphasis
- per-wave dance in the cached left/right balance and animation depth

## Approximate effects and limitations

The preview arrays are intentionally tiny and display-oriented. They are representative rather than sample-identical to the final rendered sound:

- The preview uses a short fixed visual cycle count so the shape remains readable in a small widget.
- It does not run Paulstretch, normalization, or final audio post-processing.
- Pitch evolution is not rendered at full audio resolution in the mini previews.
- Dance is represented as a compact left/right pan motion and visual animation cue, not a full-length render.
